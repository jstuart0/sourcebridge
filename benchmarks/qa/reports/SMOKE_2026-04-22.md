# Smoke-test validation — 2026-04-22

Phase-4 infrastructure smoke before the first real run. Validates
that every component in the pipeline parses, compiles, and emits
the expected shape — without needing a running server, a worker, or
an LLM API key.

## Components validated

| Component | Check | Result |
|-----------|-------|--------|
| `questions.yaml` | YAML load + class distribution + unique IDs + all fields populated | PASS (120 Q) |
| `cmd/runner` | `go build` + runs against `questions.yaml` and writes `run.jsonl` + `environment.yaml` | PASS (120 samples serialized) |
| `judge.py` | `py_compile` + argparse, anthropic import available | PASS |
| `report.py` | Synthetic paired input → valid `report.md` with Decision Rule block | PASS |
| `seed.sh` | `bash -n` syntax check | PASS |

## What is NOT proven by this smoke

- Real baseline arm (requires `workers/` + `uv` + an LLM for the
  Python subprocess; failed in this smoke because `workers-dir` was
  set to `/nonexistent` on purpose).
- Real candidate arm (requires a running server with
  `SOURCEBRIDGE_QA_SERVER_SIDE_ENABLED=true` + an indexed repo with
  the corresponding ID).
- Real judge calls (requires `ANTHROPIC_API_KEY`).
