WARNING: Back up your encryption Secret before every helm upgrade.

  The `{{ include "sourcebridge.fullname" . }}-secrets` Secret contains the
  encryption key for all stored LLM API keys and git PATs. If this Secret is
  manually deleted between upgrades, `helm upgrade` generates a new random key
  and ALL stored credentials become permanently unreadable.

  Before every upgrade:
    kubectl get secret {{ include "sourcebridge.fullname" . }}-secrets \
      -n {{ .Release.Namespace }} \
      -o yaml > sourcebridge-secrets-backup-$(date +%Y%m%d).yaml

  The `helm.sh/resource-policy: keep` annotation prevents Helm from deleting
  the Secret on uninstall, but does NOT prevent operators from deleting it
  manually.

== Worker Memory ==

  The default worker memory limit is 2Gi. For production workloads with
  higher concurrency (SOURCEBRIDGE_KNOWLEDGE_MAX_CONCURRENCY > 1) or large
  model context windows, increase the limit explicitly:

    helm upgrade {{ .Release.Name }} sourcebridge/ \
      --set worker.resources.limits.memory=4Gi \
      -n {{ .Release.Namespace }}

  Changing this value triggers a Deployment rollout. Size the limit to at
  least 2× the HuggingFace model cache footprint for your configured models.

== Prometheus Metrics ==

  /metrics is NOT exposed via the Ingress (CA-318). Prometheus should scrape
  the API's metrics endpoint via the in-cluster Service using a ServiceMonitor:

    kubectl apply -f deploy/kubernetes/base/api-servicemonitor.yaml

  Adjust the ServiceMonitor's selector labels to match your Prometheus Operator
  instance before applying.

== NetworkPolicy Hardening ==

{{- if .Values.networkPolicy.enabled }}

  NetworkPolicy hardening is ENABLED. A default-deny-all policy plus a
  curated allow-set is active for this namespace.

  WARNING: The bundled allow-set covers standard SourceBridge traffic only
  (API ↔ SurrealDB, API ↔ worker gRPC, API ↔ Redis, web → API proxy,
  Traefik → API/web, all pods → kube-dns, API → external HTTPS/HTTP/SSH,
  worker → external HTTPS/HTTP). If you have additional traffic patterns
  (e.g. a cluster-internal Ollama, a Prometheus scraper, external database
  connections) you must add supplementary NetworkPolicy resources alongside
  this chart or the traffic will be silently dropped.

  To disable:
    helm upgrade {{ .Release.Name }} sourcebridge/ \
      --set networkPolicy.enabled=false \
      -n {{ .Release.Namespace }}

{{- else }}

  NetworkPolicy hardening is disabled (default). To enable the default-deny
  + allow-set policies, run:

    helm upgrade {{ .Release.Name }} sourcebridge/ \
      --set networkPolicy.enabled=true \
      -n {{ .Release.Namespace }}

  Read the caveats above (enable with enabled=true) before toggling on
  existing clusters — the allow-set may need operator-specific additions.

{{- end }}

    helm upgrade {{ .Release.Name }} sourcebridge/ \
      --set securityContext.readOnlyRootFilesystem=false \
      -n {{ .Release.Namespace }}

  readOnlyRootFilesystem hardening is disabled (default). To enable, run:

    helm upgrade {{ .Release.Name }} sourcebridge/ \
      --set securityContext.readOnlyRootFilesystem=true \
      -n {{ .Release.Namespace }}

  The chart auto-provisions emptyDir volumes for the cache paths each
  container needs (/tmp on all three, /var/cache/sourcebridge on the
  worker, /app/.next/cache on the web). Verify the rendered output
  before applying to a long-lived cluster (CA-322).

{{- end }}
